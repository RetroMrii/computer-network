What just happened?

Each client was handled independently

The simulator showed several clients connecting to the server at almost the same time. Instead of making every client wait in one long line, the server accepted each TCP connection and assigned it to a separate worker thread.

1. TCP connection opened

Each browser/client first opened a TCP connection to the server socket. The server was already listening on its configured host and port.

2. The server accepted the client

The server's accept loop received the connection. In the real Python code, each accepted client socket is passed to a separate thread.

3. HTTP request was read

The worker thread read raw bytes from the socket until the HTTP header ended with the required blank-line marker: \r\n\r\n.

4. The response was returned

After parsing the request and finding the static file, the server built an HTTP/1.0 response with headers such as Content-Length and Content-Type, sent the file bytes, and closed the connection.

The key idea is concurrency: a slow or busy client should not freeze the entire server. Each connection gets handled separately, so multiple clients can be served during the same time window.
